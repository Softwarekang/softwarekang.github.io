Eventhub Details

2022-06-02

介绍

Eventhub 是一种无服务器事件总线，可使用从您的应用程序生成的事件，更轻松地大规模构建事件驱动型应用程序。EventBridge 提供从事件源（例如 TAPD ）到目标对象（例如 Faas、SaaS ）的实时数据流。您可以设置路由规则，以确定将数据发送到何处，从而构建在事件发布者和使用者完全解耦的情况下对数据源进行实时响应的应用程序架构。

优势

构建事件驱动型架构

EventHub 简化了构建事件驱动型架构的流程。借助 EventHub，您的事件目标不需要知道事件源，因为您可以直接进行筛选并发布到 EventHub。无需任何设置。通过松散耦合的事件驱动型架构提高开发人员敏捷性以及应用程序弹性。

消息引擎能力

  • Eventhub 具备事件源、事件规则、事件目标等概念。同时具备常见消息引擎（kafka、rocketmq)能力，在 push/sub 场景下同样适用。
  • 提供 Go SDK 支持生产、消费事件、延时事件、死信队列、消息重放、tag 过滤、request/reply 能力。

快速连接 SaaS 应用

EventHub 从受支持的 SaaS 应用程序中提取数据，并将其发送到 SaaS 目标（通过 API 目标 – 事件的一种 HTTP 调用终端节点目标），而且不需要编写自定义集成代码。您可以使用 EventHub 连接您的 SaaS 应用程序，或使用 SaaS 应用程序中的事件来触发相应业务的工作流程。

安全性 & 可观测与可治理能力

  • 支持HTTP/HTTPS协议。
  • 提供事件的可观测性能力，支持事件查询、审计以及全链路的追踪；
  • 提供事件的可治理能力，支持事件流控、事件回放、以及事件重试策略等；

高性能&高可用

  • 服务可用性 99.95%，Region 化、多可用区、分布式集群化部署，具备极强的容灾能力。

架构设计

事件源：

  • 事件的来源，负责生产事件。

事件总线：

  • 事件的中转站，负责事件的中间转储。

事件规则：

  • 用于监控特定类型的事件。当发生匹配事件时，事件会被路由到与事件规则关联的事件目标。

事件目标：

  • 事件的处理终端，负责消费事件。

Schema：

  • 定义事件的结构。针对发送到自定义事件总线的事件，您可以自定义 Schema，也可以通过过滤事件，推导出事件结构从而得到 Schema。以上 Schema 都存储在 Schema 注册表中，事件目标可以从 Schema 注册表中下载事件 Schema 的 Code Binding，实现快速访问事件。
